## 1. 文書概要

### 1.1 目的

本書は、既存設計に基づく Phase1 実装でセキュリティ要件の漏れを防ぐため、
横断チェックリストを定義する。

### 1.2 対象範囲

- Cookie session
- session fixation
- CSRF
- CORS
- RBAC（admin/viewer）
- file upload validation
- path traversal / XSS / prompt injection / SSRF
- PII / raw text logging 禁止
- external API 送信範囲
- Qdrant payload
- audit log
- error response leakage
- request_id
- secrets
- rate limit
- dependency / supply chain
- frontend trace ID 非表示
- debug view admin限定

### 1.3 非対象

- 本番SOC運用
- WAF/IDS の詳細実装

## 2. チェック運用ルール

- 各項目は `Design / Implementation / Test` の3段階で確認する
- `FAIL` は次工程へ進めない
- `RISK_ACCEPTED` は要人間判断で記録する

---

## 3. 認証・セッション

## 3.1 Cookie session

- [ ] session token は HttpOnly / Secure / SameSite 方針に従う
- [ ] DB には token hash のみ保存する
- [ ] session 失効（logout/revoke/expiry）が機能する

## 3.2 Session Fixation

- [ ] login 成功時に pre-auth 状態を失効し session-bound 状態へ切替
- [ ] session id 再発行方針が明記されている

## 3.3 CSRF

- [ ] state-changing API で CSRF header 必須
- [ ] CSRF token refresh は single-flight で競合回避
- [ ] `/auth/csrf` レスポンスに `Cache-Control: no-store`

---

## 4. 認可（RBAC）

- [ ] admin/viewer の操作境界が API と UI で一致
- [ ] admin 専用 endpoint は viewer で 403
- [ ] debug view は admin のみ表示
- [ ] frontend に内部 trace id を viewer へ露出しない

---

## 5. CORS / リクエスト境界

- [ ] CORS origin は環境ごとに明示許可のみ
- [ ] wildcard 許可は Phase1 ローカル以外で禁止
- [ ] credential 付きリクエストの扱いが明示される

---

## 6. ファイルアップロード

- [ ] 拡張子だけでなく MIME / magic bytes を検証
- [ ] サイズ上限を enforce
- [ ] path traversal（`../` 等）を拒否
- [ ] storage key はサーバー生成で固定

---

## 7. 出力・表示攻撃

## 7.1 XSS

- [ ] ユーザー入力/文書由来文字列は表示時に適切にエスケープ
- [ ] markdown/render で script 実行を許可しない

## 7.2 Prompt Injection

- [ ] context は untrusted data として扱う
- [ ] context 内命令を実行しない
- [ ] unknown marker は保存せず、valid marker 0件時のみ failure

## 7.3 SSRF

- [ ] 外部 URL 取得機能がある場合、許可先制限（Phase1 では原則無効）
- [ ] 内部ネットワーク到達を防ぐ

---

## 8. ログ・監査

## 8.1 ログ禁止項目

- [ ] raw document text
- [ ] raw chunk text
- [ ] raw prompt
- [ ] raw query
- [ ] raw LLM response
- [ ] PII
- [ ] secret/token/password

## 8.2 許可項目

- [ ] request_id
- [ ] error_code
- [ ] trace 用 ID（必要最小限）
- [ ] token usage summary（数値）

## 8.3 Audit Log

- [ ] actor/action/target/request_id が保存される
- [ ] target_id NULL 許容イベントの扱いが統一される
- [ ] 監査ログに機密本文を保存しない

---

## 9. API エラー・情報漏洩

- [ ] エラー応答に内部 stack trace を出さない
- [ ] `error_code` は設計済み一覧のみ返す
- [ ] validation error に過剰内部情報を含めない
- [ ] request_id を error response に含める

---

## 10. RAG / Worker 固有

## 10.1 RAG

- [ ] no_context（422）と citation_build_failed（500）の境界が実装一致
- [ ] citation は retrieval_run_items 由来のみ
- [ ] archive 文書は retrieval 対象外

## 10.2 Worker

- [ ] lease ownership 失効時に terminal update を行わない
- [ ] reclaim/retry で cleanup 方針が守られる
- [ ] Qdrant cleanup 失敗時 error_code マッピング一致

---

## 11. 外部 API / データ送信範囲

- [ ] 送信するのは最小限 context のみ
- [ ] 個人情報や秘密情報を送信しない
- [ ] 送信先モデル/サービスを構成で固定

---

## 12. Secret / 依存管理

- [ ] `.env` は commit しない
- [ ] `.env.example` に実値を載せない
- [ ] 依存ライブラリの脆弱性スキャン方針がある
- [ ] lockfile 管理を行う

---

## 13. Rate Limit / Abuse 対策

- [ ] login / rag/ask / upload / retry にレート制限方針がある
- [ ] burst と sustained の基準がある
- [ ] 過負荷時のエラー契約が定義される

---

## 14. テスト対応表

- Cookie/CSRF/RBAC: API test + E2E
- Upload validation/path traversal: API test
- Prompt injection/no_context/citation: RAG integration test
- lease/reclaim/cleanup: Worker test
- logging禁止: security test
