# PR-36 Citation Navigation

## Purpose

Citation navigation lets a user move from a chat citation to a safe source locator.
It is designed for Phase2 demos where the important path is:

```text
citation -> source locator -> bounded preview -> admin document/version context
```

## API

```http
GET /api/v1/rag/citations/{citation_id}/source
```

- Authentication required.
- Admins can resolve any citation source.
- Viewers can resolve only citations that belong to their own chat sessions.
- Ownership mismatches return `404`, not cross-user metadata.
- The response contains bounded source preview and safe locator metadata only.

## Source Locator Fields

The locator can describe:

- logical document id
- document version id
- document chunk id
- source label
- upload or external URL source type
- redacted external URL
- page range
- section title
- Excel sheet and row range
- PowerPoint slide/title
- HTML heading path
- XML path
- old-version flag

Missing metadata is returned as `null`; callers should display a generic chunk locator.

## Redaction

The source locator does not return raw file content, full chunk text, full context,
storage paths, or raw URL query strings. Preview text is length-limited and redacts
secret-like assignments, URLs, and email addresses.

External links are display-safe URLs generated by the URL ingest redaction layer.
The frontend opens them with `rel="noopener noreferrer"`.

## Frontend Behavior

- Chat citations include a "View source" action.
- Viewers see only the bounded source preview.
- Admins additionally see a document detail deep link.
- Old version warnings continue to use the existing old source badge.

## Known Limitations

- The preview is text-only.
- No PDF page image rendering.
- No OCR coordinate navigation.
- No recursive web navigation or crawler behavior.
