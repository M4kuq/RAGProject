## 4. PR 分解（推奨）

## 4.1 PR-01 scaffold / settings

- 目的: backend/frontend/worker の最小起動骨格
- 入力: 基本設計書、backend/frontend 詳細設計
- 変更対象: 設定読み込み、ディレクトリ、依存管理
- 完了条件: 起動チェック通過
- テスト条件: smoke 起動確認

## 4.2 PR-02 Alembic base (0001-0003)

- 目的: auth/chat/documents 基盤 migration
- 入力: DDL v1.8、migration設計書
- 完了条件: `upgrade head`（対象 revision）成功
- テスト条件: 制約 smoke

## 4.3 PR-03 Alembic advanced (0004-0010)

- 目的: jobs/retrieval/evaluation/audit/index/seed
- 入力: DDL v1.8、migration設計書
- 完了条件: full upgrade/downgrade 成功
- テスト条件: `upgrade -> downgrade -> upgrade`

PR-03 内部検証順（固定）:
1. 0004 jobs
2. 0005 retrieval_runs / retrieval_run_items / citations
3. 0006 evaluation
4. 0007 audit / settings
5. 0008 circular FK
6. 0009 indexes
7. 0010 seed
8. full migration cycle（upgrade -> downgrade -> upgrade）

補足:
- より安全に分割する場合は `PR-03a(0004-0007)` と `PR-03b(0008-0010)` に分割可能。
- ただし本書の標準順序は PR番号固定を優先し、PR-03 内部検証順でリスクを吸収する。

- 目的: テスト設計書 v1.0 のカテゴリを CI に段階的に組み込む
- 入力: テスト設計書
- 完了条件:
  - Tier 1 を全PR必須カテゴリとして CI 組み込み
  - Tier 2 を main merge前必須または該当PR必須として CI 組み込み
  - Tier 3 を nightly / manual / release前実行として定義
- テスト条件: migration + API contract + worker + RAG pipeline + selected E2E

CI Tier 定義:
- Tier 1（全PR必須）: lint / format / type / unit / repository / service / frontend unit-hook / OpenAPI lint
- Tier 2（main merge前または該当PR必須）: migration / API contract / DB constraint / RAG pipeline / Worker-Job / security basic
- Tier 3（nightly-manual）: E2E / long-running concurrency / Qdrant integration / LLM依存 / full demo scenario

## 5. 各 PR 共通の完了条件

- 対象設計書の要件をすべて満たす
- 追加した変更範囲のテストが通る
- セキュリティ禁止事項（raw text/prompt/PII/secret非ログ）を満たす
- 追随事項を PR 説明に明記

---

## 6. レビュー観点（PR共通）

- requirements: 要件逸脱がない
- architecture: 層責務・境界が崩れていない
- security: CSRF/RBAC/PII
- test: 変更に対応する最小十分なテスト
- documentation: 設計との差分説明

---

## 7. 停止条件（要人間判断）

- 既存確定設計と衝突する実装判断が必要
- DDL/API/RAG/Worker の矛盾が顕在化
- OpenAPI 契約確定前に実装を進める必要が出る
- 認証認可境界の仕様不足が発覚
